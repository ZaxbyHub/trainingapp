# v2.2.1 — GUI Freeze Fix & Memory Check Improvement

## What changed

### Bug Fixes
- **GUI freeze on first query**: Moved `_ensure_llm()` from main UI thread into background query thread in `app_gui.py`. Previously, llama-cpp model loading blocked the tkinter event loop synchronously, freezing the GUI. Error handling now uses thread-safe `message_queue.put()` instead of `messagebox.showerror()`.
- **Memory check now model-size-based**: Replaced unrealistic 75%-free-RAM check with `file_size * 4` requirement in `llm_interface.py`. The old check failed on most systems (requiring 75% of RAM free). The new check properly accounts for GGUF weights + KV cache + overhead based on actual model file size.

### Tests
- Behavioral tests for `_ensure_llm()` thread placement (source inspection replaced with runtime verification)
- Thread-safe error handling verification
- Memory budget tests updated for 4x multiplier

## Why

Code review identified two issues:
1. `_ask_question()` called `_ensure_llm()` on the main UI thread before spawning the worker thread, causing GUI freeze during model initialization
2. Memory check required 75% of total RAM free — impossible on most real systems with typical RAM usage

## Known caveats

- The 4x multiplier is conservative but safe for all context window sizes (covers weights + KV cache at 32K n_ctx)
