## What changed

### Chores
- **Dead code removal**: Deleted `web_ui/src/pages/MlSpikePage.tsx` and `web_ui/src/lib/ml-spike/spike-utils.ts` ŌĆö orphaned ML spike artifacts not routed or imported by any active code
- **TODO comment cleanup**: Removed 2 "TODO: verify API" comments from `web_ui/src/lib/llm/web-llm-service.ts` (CreateMLCEngine API has been verified in production use)
- **Unused React imports**: Removed `import React from 'react'` from 5 source files where Vite's `jsx: react-jsx` transform makes it unnecessary (ChatPage.tsx, DocumentsPage.tsx, ErrorBoundary.tsx, InferenceModeToggle.tsx, Toast.tsx). Also removed unused `ToastProps` interface in Toast.tsx
- **Unused imports in task44 verification test**: Removed unused `afterEach` and `vi` from vitest import in `web_ui/src/test/verification/text-chunker.task44.test.ts`

### Fixes
- **Unused destructured variable**: Removed `isModelReady` from `web_ui/src/pages/SettingsPage.tsx` destructuring (was unused, blocked TypeScript strict build)
- **TypeScript strict mode in SettingsPage tests**: Added missing mock properties (`setModelReady`, `setModelLoadingProgress`, `modelLoadingProgress`, `modeError`, `isDark`) across 7 `mockReturnValue` calls in `web_ui/src/pages/SettingsPage.test.tsx`. Prefixed 3 unused parameters with underscores. Fixed 5 IDBRequest type casts using `as unknown as` pattern
- **TypeScript strict mode in ChatPage.test.tsx**: Removed unused `act` import, corrected ChatPage import path from `'../ChatPage'` to `'./ChatPage'`, removed broken `vi.mocked+dispose` test block (used invalid pattern), added `async` to 2 `it()` callbacks containing `await`, fixed unused `setTimeoutSpy` const
- **TypeScript strict mode in DocumentsPage tests**: Removed unused React imports, prefixed 5 unused mock callback params with underscores, removed invalid number `1` as 3rd arg from 2 `chunkText()` calls, updated mock `TextChunker.chunkText` signature

### Tests
- **SSEStreamConsumer test suite** (26 tests): New `web_ui/src/lib/api/streaming.test.ts` covering constructor, callback registration, fetch contract (POST, Content-Type, Authorization/Bearer, JSON body, AbortController), error paths (non-OK, null body, AbortError), SSE line parsing (token/done/error events, malformed JSON, non-data lines), stop() safety (idempotent), and readStream chunking with line buffering
- **ApiClient test suite** (29 tests): New `web_ui/src/lib/api/client.test.ts` covering ApiError class, constructor (default baseUrl, trailing slash, optional token), all 9 API methods (listDocuments, uploadFile, uploadBatch, ingestDirectory, clearDocuments, ask, search, getSettings, updateSettings, getStats), offline detection, non-OK responses, auth token injection (constructor token precedence over getToken), FormData multipart, and parseErrorResponse (JSON + non-JSON fallback)

## Why

The 8-phase Document Q&A SPA implementation was complete and the production build was passing. Final council flagged 11 advisory findings, of which 4 needed follow-up:
1. Dead code: MlSpikePage and spike-utils still in tree
2. Web-llm-service TODO comments questioning API verification
3. TypeScript strict mode errors across multiple test and source files
4. Zero test coverage for SSEStreamConsumer and ApiClient

These were addressed in Phase 9 of the plan through 10 individual tasks, each going through the full Stage A + Stage B QA gate pipeline (declare_scope ŌåÆ coder delegation ŌåÆ automated gates ŌåÆ reviewer + test_engineer).
