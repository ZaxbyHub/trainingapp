# Release v0.2.0 — UI/UX Modernization (Windows 11 Fluent Design)

## What changed

### Navigation and layout (feat)
- Replaced single-pane layout with a Windows 11 Fluent Design navigation rail
  (Chat, Documents, Settings, Help pages).
- Navigation switching via rail buttons or keyboard shortcuts (`Ctrl+,` → Settings).
- Empty state moved out of `CTkScrollableFrame` into a sibling start surface with
  primary CTA (Add Documents) and secondary CTA (Open Documents Page); hidden once
  chat has messages.

### Chat composer (feat)
- Message composer upgraded to `CTkTextbox` (multiline); `Enter` inserts a newline,
  `Ctrl+Enter` submits.
- `Escape` clears/cancels via `_handle_escape`.
- Copy button added to every assistant message bubble.
- Retrieved chunks shown in a collapsible expander per message.

### Documents page (feat)
- Full document library with command row: Add Folder, Add Files, Clear All.
- Per-document display: name, type/extension, chunk count, indexed time, source path.
- Per-document actions: Details dialog, Delete (with confirmation).
- "Add Files" uses `filedialog.askopenfilenames` + `engine.ingest_file` per file;
  "Add Folder" uses `filedialog.askdirectory` + `engine.ingest_directory`.
- List refreshes after every ingest/delete/clear-all operation.

### Settings page (feat)
- Inline settings page replaces the legacy modal as primary path (`Ctrl+,` now
  switches to the Settings page rather than opening a dialog).
- All 19 canonical fields exposed across four grouped sections:
  Basic, Retrieval, Reranking, and Model/Backend.
- Fast / Balanced / Quality preset buttons update form fields without saving.
- `min_similarity` included in all three preset dicts (Fast: 0.35, Balanced: 0.30,
  Quality: 0.25).
- Full validation before save: `chunk_overlap < chunk_size`, similarity/temperature/
  token/context bounds, n_ctx/n_threads positive bounds.
- `reranking_enabled` UI default is `False` everywhere (was `True` in legacy
  `SettingsDialog._create_widgets`).

### Help page (feat)
- Runtime status section: app version (`DocumentQAApp.VERSION`), GGUF model path,
  ChromaDB path, log path, document count (live via `_refresh_help_status`).
- Keyboard shortcuts reference table.
- How-to workflow steps.

### Progress visibility (fix)
- `progress_frame` is no longer packed at startup; introduced `_show_progress(stage,
  progress)` and `_hide_progress()` helpers called during ingest/query cycles.

### Version single source of truth (fix)
- Removed `APP_VERSION = "1.0.0"` module constant; `DocumentQAApp.VERSION = "2.0.0"`
  is the sole authoritative version string. Help page now reads `self.VERSION`.

### Reranking default safety (fix)
- `SettingsDialog._create_widgets` was initialising the reranking toggle with
  `True` as the default when the key was absent. Changed to `False`, matching
  `_populate_fields` and all inline settings paths.

## Why
- Initial prototype navigation shell did not satisfy the full modernization spec.
- Settings page exposed only a subset of canonical fields.
- Reranking could be silently re-enabled on minimum-hardware machines.
- Legacy `APP_VERSION` constant and class `VERSION` attribute were out of sync.
- BM25 pipeline test was failing in environments without `rank_bm25` installed.

## Migration steps
No API or config key changes. Existing `app_settings.json` files are read via
`normalize_settings()`, which migrates `rag_*`-prefixed legacy keys to canonical
form automatically. No manual migration needed.

## Breaking changes
None. The legacy `SettingsDialog` modal is still accessible via `_open_settings()`
for any code paths that reference it directly; only the `Ctrl+,` shortcut has been
rebound to the inline Settings page.

## Known caveats
- All UI tests skip in headless environments where `tkinter`/`customtkinter` is
  unavailable (CI with no display). This is expected; the tests use
  `inspect.getsource` and will run when a display is present.
- `test_query_matches_document_with_period` skips when `rank_bm25` is not installed;
  the remaining four BM25 tokenizer tests run in all environments.
- The inline Settings page does not yet persist `embedding_model` or `reranker_model`
  (both are displayed read-only); changing them requires editing `app_settings.json`
  directly and restarting.
